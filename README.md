Forge WD Browser 🏷️A high-performance image interrogation and tagging extension for Stable Diffusion WebUI Forge, inspired by the UI/UX of the Civitai Browser and powered by SmilingWolf's WD Tagger.✨ FeaturesCivitai-Style Interface: A modern, split-pane UI. Browse your local generations on the left; view and edit interrogated tags on the right.Optimized for Forge: Built to utilize the onnxruntime backend for near-instant tagging without interfering with Forge's VRAM management.Latest Models: Support for WD SwinV2 V3, WD ViT V3, and legacy V2 models.Batch Processing: Interrogate entire folders of images and automatically save .txt sidecar files for LoRA training or archival.Smart Filtering: * Adjustable confidence thresholds for general and character tags.Blacklist system to filter out unwanted tags (e.g., "rating: safe", "monochrome").Optional escape characters for parentheses (Booru-style).🚀 InstallationOpen SD WebUI Forge.Navigate to the Extensions tab -> Install from URL.Paste the URL of this repository:https://github.com/YOUR_USERNAME/forge-wd-browserClick Install.Restart UI (or close and relaunch the terminal).Note: On first run, the extension will download the required ONNX model files (~100-200MB) from Hugging Face.🛠️ How to UseSingle Image ModeGo to the WD Browser tab.Select an image from your outputs gallery or drag-and-drop a file.Adjust the Threshold (Default: 0.35).Click Interrogate.Use the Send to Prompt buttons to instantly move tags to txt2img or img2img.Batch ModeSwitch to the Batch Interrogator sub-tab.Input your source folder path.Configure your tag suffix (e.g., .txt).Click Run Batch Interrogation.📊 ComparisonFeatureStandard WD14 TaggerForge WD BrowserUI LayoutSingle ColumnCivitai-style GalleryBackendTensorFlow/ONNXOptimized ONNXIntegrationStandaloneDeep Forge HookMass TaggingBasicAdvanced w/ Preview🤝 CreditsUI Inspiration: eduardoabreu81's Civitai BrowserTagging Logic: SmilingWolf for the incredible WD Tagger models.Platform: lllyasviel for SD WebUI Forge.
